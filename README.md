# A 股选股打分模型

基于"股票涨跌理论"的命令行打分工具。

把理论的 **G（关注度）/ Z（确定性）/ K（盈亏比）/ S（最大可卖量）** 四变量和
**六大判断依据**翻译成 akshare 可抓取的信号，加权打分后输出 Excel 报告。

## 安装

```bash
cd stock_scorer
pip3 install -r requirements.txt
```

依赖：`akshare`、`pandas`、`openpyxl`。无需任何 token。

## 快速开始

```bash
# 首次：生成示例记忆库模板
python3 run.py --init-memory

# 每日盘后运行
python3 run.py

# 强制刷新缓存
python3 run.py --no-cache

# 指定市场环境 / 候选数
python3 run.py --regime bull --top 100
```

输出：`ranked_candidates.xlsx`，含四个 sheet：
- **排名**：总分降序的候选清单，附六大依据子分、各 note、K/PE/流通市值等
- **权重**：本次实际归一后的权重矩阵（含跃迁加权）
- **风控**：从 `config.RISK` 拷贝的纪律表
- **元数据**：生成时间、市场环境、候选数、门槛分

## 字段说明

| 列 | 含义 |
|---|---|
| `Z1_业绩向好性` | 0~2，营收同比 ≥ 20% 加 1，净利同比 ≥ 30% 加 1 |
| `Z2_同质同价` | 0~2，PE < 15 记 2，15~25 记 1，> 60 记 0（粗替代） |
| `Z3_形态` | 0~2，MA 多头 +1，超跌 +1 |
| `Z4_市场关注度` | 0~2，换手率 > 5% +1，上龙虎榜 +1 |
| `Z5_最大可卖量` | 0~2，流通市值 < 80 亿 +1，30 日内有解禁 -1 |
| `Z6_特定情境记忆` | 0~2，从 `memory_seed.xlsx` 衰减后预期涨幅推算 |
| `K_盈亏比` | 近 60 日 (高点距离 / 低点距离) 比值 |
| `G_量比` | 当日量 / 5 日均量，≥ 1.5 触发择时 |
| `timing_ok` | 量比信号是否满足 |
| `总分` | 六大子分按权重加权和，乘 5 归一到 0~10 |

## 与方案七步的对应关系

| 方案步骤 | 对应文件 / 函数 |
|---|---|
| 1. 股票池剔除 | `filters.apply_hard_filters` |
| 2. G/Z/K/S 信号映射 | `signals.compute_signals` |
| 3. 加权打分 | `scorer.score_all` |
| 4. 择时（R 值方向） | `signals` 中的 `G_量比` / `timing_ok` |
| 5. 仓位与风控 | `config.RISK`，输出"风控"sheet |
| 6. 卖出依据跃迁 | `scorer.leap_boost` |
| 7. 特定情境记忆库 | `memory.load` / `memory.match` |

## 记忆库

`memory_seed.xlsx` schema（自行编辑与维护）：

| 题材 | P1日期 | P1涨幅 | P2日期 | P2涨幅 | P3日期 | P3涨幅 | 关联代码 |
|---|---|---|---|---|---|---|---|

衰减公式：`贡献 = Σ(涨幅 × P权重 × 0.5^(距今天数/60))`，权重 P1=50% / P2=30% / P3=15% / P4=3%。

## 常见问题

- **运行慢**：候选数控制在 50~100，已默认有 12 小时文件缓存。首次跑会拉所有候选的日 K + 财务，约 5~10 分钟。
- **akshare 接口偶发 403**：换网络重试或加大缓存时长（`config.CACHE_TTL_HOURS`）。
- **PE/PB 为空**：新股或亏损股，会被自动剔除 z2 加分。

## 与原理论的偏差

为工程落地做了几处简化：
- Z2 同质同价没有现成行业 PE 中位数接口，暂以全市场绝对 PE 替代；
- Z5 最大可卖量未扣除前十大股东锁仓（季报才有），用流通市值近似；
- 特定情境记忆需要人工维护题材库，MVP 只给 seed 模板。
